Receive blank page when creating tasks in Task Manager

For other users with this issue, here is specifically what I did. I can't guarantee it will work for everyone, but it seems to have corrected it for me at least: 

gksu gedit /var/www/fog/management/includes/tasks.confirm.include.php
Search Menu > Replace
Search for: &$tmp
Replace with: $tmp
Replace All (there were somewhere around twenty instances)
Save
Close
Then try to schedule your upload task again.
Reference: http://fogproject.org/forum/threads/cannot-create-tasks-task-manager-blank.3823/#post-9593

Upload Error: Extended record needed (1040>1024), not yet supported

On FOG wiki, it provides very detailed solution to this problem, see the link below:
http://www.fogproject.org/wiki/index.php/Error:_extended_record_needed

But in practice, I found the problem usually caused by the file system inconsistency on Linux systems or there are too many fragments on Windows system. So here is how to resolve this issue:

On Linux system, boot from PXE admin option, then run the following command on Linux partition: 

fsck /dev/sda1

On Windows system, run defragmentation on the system drive
